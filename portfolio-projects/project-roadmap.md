# Roadmap de Execução dos Projetos

Este roadmap organiza a evolução dos projetos de portfólio por ordem estratégica para processos seletivos em tecnologia, dados, mercado financeiro, consultoria e beauty tech.

## Prioridade 1 — Projetos mais fortes para bancos, fintechs e consultorias

### 1. Digital Banking API

Foco: Java, Spring Boot, APIs REST, regras de negócio e banco de dados.

Entregáveis prioritários:

- API funcional.
- PostgreSQL configurado.
- Swagger.
- Testes unitários.
- README com prints e endpoints.

### 2. Credit Risk Scoring

Foco: Python, análise de dados, risco de crédito e modelo de classificação.

Entregáveis prioritários:

- Dataset simulado.
- Notebook de análise.
- Score de risco.
- Dashboard simples.
- Relatório executivo.

### 3. Financial Data Pipeline

Foco: engenharia de dados, ETL, SQL e Power BI.

Entregáveis prioritários:

- Pipeline em Python.
- PostgreSQL.
- Scripts SQL.
- Dashboard.
- Documentação de arquitetura.

## Prioridade 2 — Projetos para analytics, risco e investimentos

### 4. Fraud Detection System

Foco: análise de comportamento, regras de fraude e alertas.

### 5. Personal Investment Portfolio Analyzer

Foco: dados financeiros, carteira de investimentos e visualização.

## Prioridade 3 — Projetos Beauty Tech para diferenciar o portfólio

### 6. Beauty Sales Intelligence Dashboard

Foco: BI, e-commerce, vendas e performance.

### 7. CRM Beauty Retention Engine

Foco: CRM, retenção, recompra e segmentação.

### 8. AI Beauty Product Recommendation

Foco: produto digital, IA, recomendação e UX.

## Critério de qualidade para considerar um projeto pronto para currículo

Um projeto só deve ser destacado como pronto quando tiver:

- Código organizado.
- README completo.
- Prints ou demonstração visual.
- Dados simulados ou fonte explicada.
- Instruções de execução.
- Principais decisões técnicas documentadas.
- Próximos passos definidos.

## Estrutura ideal no GitHub

Cada projeto deve evoluir para conter:

```text
project-name
├── README.md
├── data
├── src
├── notebooks
├── docs
├── tests
└── dashboard ou frontend
```
